Everything you need to know to train great machine learning models with Lobe.

Lobe is a free, private desktop application that has everything you need to take your machine learning ideas from prototype to production.

Machine learning is software that learns to perform a task from a collection of examples rather than through a person explicitly defining rules and formulas. This learning software is called a model. Teaching a model through examples is called training. Learn more in this beginner crash course.

This version of Lobe learns to look at images using image classification - categorizing an image into a single label overall. We are working to expand to more types of problems and data in future versions.

Image classification is categorizing an image into a single label to represent its content. Apps using image classification could:
  • Tell you when someone is coming up your front steps
  • Send you photos of a new bird that just started showing up at your bird feede
  • Count the number of push-ups you’ve done in a workout
  • Alert you when a shelf is empty

Labeling is assigning categories to your images to create examples that teach Lobe. These examples are commonly known as a dataset. From this dataset, Lobe will learn to automatically predict a label for a given image.

Collecting a dataset starts with being clear about the problem you’re trying to solve. If the goal is training a model to recognize certain objects or gestures, you need examples of those things in different settings. For images, that means taking photos or videos under varying lighting, angles, and backgrounds so the model doesn’t get fooled by small changes. You’ll also want to balance the classes, meaning you shouldn’t have 500 pictures of one object and only 20 of another. Each example should be labeled correctly, because mislabeled data will confuse the model more than help it. Depending on your resources, you can gather data yourself with a camera, pull from open-source datasets online, or even generate synthetic samples with software. The key is variety, quality, and accurate labels—those three together make a dataset actually useful.

Collect images that you expect to see in the real world.
Lobe can only learn the patterns that exist in the images you provide as examples. Collect images from the same source you expect to use with your exported model.
Capture as many variations as possible.
Try to capture all the variations that naturally occur by collecting images in different conditions. Try different backgrounds, lighting, orientations, or zoom. This helps Lobe learn what parts of the image are useful for making predictions and what is noise.
Make sure your content is visible in the center square of the image.
While training your model, Lobe crops the center square of your images.

Label for each desired prediction.
Create a label for each category you want the model to predict.

Gather as many different images as you can.
CA rough guideline is 100-1,000 images per label for smaller problems. To quickly get started with your project, Lobe has a minimum requirement of 5 images per label.
Have roughly equal number of images per label.
Balance the number of images between each label. If imbalanced, Lobe will be biased to predict the labels with more images and ignore the labels with fewer images
